★★★ 必关注

lfnovo/open-notebook

  • Star: 26,811 total / +794 today
  • 语言: TypeScript
  • 是什么：开源版 Notebook LM，支持上传文档、URL、文本并生成交互式 AI 笔记，功能更灵活
  • 为什么火：Google 的 NotebookLM 定义了”AI 文档助手”品类但始终封闭，open-notebook 给出了完全可控的开源替代方案——支持本地部署、自定义模型、更多输出格式。794 star 的日增说明社区对这一品类的需求远未被满足。
  • 适合谁：需要私有化 AI 知识管理的个人和团队；关注 RAG 落地产品的开发者

obra/superpowers

  • Star: 219,955 total / +700 today
  • 语言: Shell
  • 是什么：Agentic 技能框架 + 软件开发方法论，定义了 Agent 如何通过可组合技能完成复杂任务
  • 为什么火：22 万 star 的庞大体量 + 700 日增表明它正在成为 Agent 开发的”事实标准”。它将”技能”定义为可组合、可共享的原子单元，这套理念正在被 Claude Code Skills、Cursor Rules 等工具生态广泛借鉴。不是 AI 工具，而是定义 Agent 如何工作的元框架。
  • 适合谁：所有做 Agent 开发的团队；关注 AI 工程化方法论的技术负责人

Panniantong/Agent-Reach

  • Star: 22,749 total / +683 today
  • 语言: Python
  • 是什么：给 AI Agent 装上”眼睛”——让 Agent 能读取 Twitter、Reddit、YouTube、GitHub、Bilibili、小红书等主流平台
  • 为什么火：Agent 目前最大的能力短板是无法感知互联网上正在发生什么。Agent-Reach 精准打中这个痛点：将多平台内容采集、清洗、结构化输入给 Agent。支持 Bilibili 和小红书意味着它对中国开发者生态有天然友好度。中国团队出品。
  • 适合谁：做社交监听、竞品分析、舆情追踪的 Agent 开发者；需要 Agent 具备”实时互联网感知”能力的团队

CopilotKit/CopilotKit

  • Star: 33,456 total / +631 today
  • 语言: TypeScript
  • 是什么：Agent 与 Generative UI 的前端基础设施，支持 React、Angular、Mobile、Slack 等多端集成
  • 为什么火：Agent 不能只活在 CLI 里。CopilotKit 解决了”Agent 如何与用户交互”这一层——提供开箱即用的聊天 UI、流式渲染、工具调用可视化等组件。631 star 日增说明前端开发者正在大规模涌入 Agent 赛道。
  • 适合谁：需要为 Agent 产品构建前端界面的全栈团队
